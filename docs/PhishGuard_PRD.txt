Section 1 — Product Requirements Document (PRD)
1.1 Product Overview
PhishGuard is a real-time, AI-powered phishing detection and protection system that combines deep learning classifiers with LLM-based contextual reasoning. Users submit any URL and receive a verdict (Safe / Suspicious / Malicious), a risk score (0–100), and a human-readable explanation generated by a Gemini-powered cyber analyst.

Section 2 — System Architecture Document
2.1 Architecture Overview
PhishGuard follows a 3-tier architecture: a React frontend, a FastAPI backend, and a data/ML layer. The ML inference and LLM call are made asynchronously, with results fused by the Hybrid Decision Engine before returning to the client.

2.3 Data Flow
User enters URL in React frontend and clicks 'Scan'
Frontend sends POST /api/v1/scan with { url: string }
FastAPI validates URL format, returns 400 if invalid
URL Feature Extractor computes lexical + domain features
Tokeniser converts URL to token IDs for transformer input
DistilBERT classifier infers phishing probability (async task)
Gemini API call sends URL + features for contextual analysis (async task)
Both async tasks run concurrently using asyncio.gather()
Hybrid Decision Engine fuses scores into final verdict + risk score
Result stored in PostgreSQL (URL hash, verdict, score, timestamp)
Response returned to frontend: { verdict, score, explanation, features }
Frontend renders verdict badge, risk gauge, explanation, feature list

5.4 Key Implementation Notes for AI Builder
Model Loading: Load DistilBERT model ONCE at FastAPI startup using lifespan context manager, not per-request, to avoid 3+ second cold start on every scan.
Async Concurrency: Use asyncio.gather() to run DL inference and Gemini API call concurrently, cutting total latency roughly in half.
URL Hashing: Store SHA-256(url.lower().strip()) in DB to deduplicate and enable cache lookups without storing raw URLs.
Result Caching: Cache verdicts in Redis with 1-hour TTL using url_hash as key. Return cached result immediately on repeat scans.
Gemini JSON Mode: Always instruct Gemini to respond ONLY with valid JSON. Parse with try/except and fall back to LLM-unavailable response on parse failure.
Feature Extraction: tldextract is essential for reliable TLD and subdomain extraction. Do not use urllib.parse alone.
Frontend State: Use React Query (TanStack Query) for scan API calls to get built-in loading/error states and caching.
Error Boundaries: Wrap main result area in React Error Boundary so a crashed component does not blank the whole page.
Docker Volumes: Mount ml/models/ directory as a read-only Docker volume so model files are not baked into the image.
Chrome Extension: Use Manifest V3 with a service worker background script. Send URL to backend via fetch() from the popup script.

Section 8 — Glossary
Term
Definition
Phishing
A cyberattack where a fraudulent website or message mimics a legitimate entity to steal credentials or sensitive data.
Zero-day URL
A malicious URL that has not yet been indexed by blacklists or seen during model training.
Homoglyph
A character visually identical or similar to another (e.g., Cyrillic 'а' vs Latin 'a') used to spoof domain names.
Typosquatting
Registering domains with common misspellings of legitimate brands (e.g., gooogle.com).
DL
Deep Learning — machine learning using multi-layer neural networks.
LLM
Large Language Model — a transformer-based model (e.g., Gemini, GPT) trained on vast text corpora.
DistilBERT
A smaller, faster version of BERT (Bidirectional Encoder Representations from Transformers) retaining 97% of BERT's performance at 60% of the size.
ROC-AUC
Receiver Operating Characteristic — Area Under Curve; measures classifier's ability to distinguish classes across all thresholds.
WordPiece
A subword tokenisation algorithm used by BERT-family models.
TLD
Top-Level Domain — the last part of a domain name (e.g., .com, .xyz, .tk).
WHOIS
A query protocol providing registration information about a domain.
FastAPI
A modern Python web framework for building REST APIs, based on OpenAPI and Pydantic.
Hybrid Decision Engine
PhishGuard's rule-based fusion module that combines DL probability and LLM risk level into a single verdict.
